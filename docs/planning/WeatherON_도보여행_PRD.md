## 1. 배경 및 문제 정의

### 현재 시장의 공백

| | 네이버·카카오 지도 | 구글 지도 | 기존 날씨 앱 |
|---|---|---|---|
| 도보 경로 안내 | △ 기본 수준 | ○ 양호 | ✗ 없음 |
| 한국 도보 코스 DB | △ 일부 | △ 약함 | ✗ 없음 |
| 코스별 구간 날씨 | ✗ | ✗ | ✗ |
| 날씨 창 + 출발 시점 추천 | ✗ | ✗ | △ 지점 한정 |
| 도보 컨디션 지수 | ✗ | ✗ | ✗ |
| 위험 기상 ↔ 코스 연결 경보 | ✗ | ✗ | △ 지점 한정 |

### 핵심 인사이트

- 도보여행은 **이동 내내** 날씨에 노출된다. "목적지 날씨"는 의미 없고, "걷는 시간 동안 구간별 날씨"가 필요하다.
- 제주 올레길, 지리산 둘레길 등 **한국 대표 장거리 도보 코스**는 국내 지도 앱에서 제대로 지원하지 않는다.
- 날씨에 따른 **출발 시점 판단**은 현재 아무 앱도 코스 단위로 제공하지 않는다.

### 타깃 사용자

- 주말 당일치기 도보 여행자 (제주 올레, 둘레길, 산책로)
- 등산·트레킹 계획자 (날씨 불확실성 높음)
- 장거리 도보 여행자 (코리아 둘레길, 해파랑길)
- 여행 중 도보 관광객

## 3. 기능 명세

### F1. 한국 도보 코스 DB

**개요**: 국내 주요 장거리·중거리 도보 코스를 내장 DB로 제공한다.

**수록 코스 (MVP 기준)**

| 코스명 | 코스 수 | 총 거리 | 데이터 출처 |
|---|---|---|---|
| 제주 올레길 | 26개 | 약 437km | 사단법인 제주올레 공식 데이터 |
| 지리산 둘레길 | 21개 구간 | 약 274km | 지리산 둘레길 공식 |
| 북한산 둘레길 | 21개 구간 | 약 71.5km | 국립공원공단 |
| 서울 둘레길 | 8개 코스 | 약 157km | 서울시 공식 |
| 해파랑길 | 50개 구간 | 약 770km | 한국관광공사 |
| 코리아 둘레길 | 선별 구간 | - | 문체부 공식 |

**각 코스 데이터 필드**

```
- 코스명, 지역
- 거리 (km), 예상 소요 시간 (분)
- 난이도: 쉬움 / 보통 / 어려움
- 경고 구간 (가파른 지형, 해안 노출, 그늘 없음 등)
- 출발지/도착지 좌표
- 경유 구간 좌표 배열 (폴리라인)
- 고도 프로파일 배열
- 시즌 적합도 (봄/여름/가을/겨울)
- 연결 교통편
```

**v2.0 추가 예정**: 사용자 직접 코스 제보, 지역별 숨은 길 큐레이션

### F2. 코스별 구간 시간대 날씨

**개요**: 단순 목적지 날씨가 아닌, 코스 경유 지점 각각에 대해 예상 통과 시각 기준 날씨를 제공한다.

**로직**

```
입력:
  - course_id          (선택한 코스)
  - start_time         (출발 예정 시각)
  - walking_speed      (slow / normal / fast → km/h 환산)

계산:
  for each waypoint in course.waypoints:
    eta = start_time + (waypoint.cumulative_distance / walking_speed)
    weather[waypoint] = forecast_at(waypoint.lat, waypoint.lon, eta)

출력:
  - 구간별 날씨 카드 (타임라인 형태)
  - "구간 N 통과 시 비 예상" 강조 표시
  - 전체 코스 날씨 요약 (최악 구간 기준)
```

**UI 표현**

- 수평 타임라인: 출발 → 구간1 → 구간2 → … → 도착
- 각 지점에 아이콘 (맑음 ☀️ / 흐림 ⛅ / 비 🌧 / 위험 ⚠️)
- 기온, 강수 확률, 바람 속도 표시
- 산악 코스: 고도별 기온 보정 적용 (-0.6°C / 100m 상승)

### F3. 도보 컨디션 지수 (WCI — Walking Condition Index)

**개요**: 기상 데이터와 코스 특성을 결합해 "이 코스, 지금 걷기 어떤가요?"를 5단계로 표현한다.

**산출 공식**

```
WCI = w1×체감온도점수 + w2×UV점수 + w3×바람점수 + w4×강수점수 + w5×코스보정

체감온도점수:
  체감 10~22°C → 100점
  체감 23~28°C → 70점 (덥지만 가능)
  체감 29~33°C → 40점 (주의)
  체감 34°C 이상 → 10점 (위험)
  체감 5°C 이하 → 30점 (방한 필요)

UV점수:
  UV 0~2 → 100점
  UV 3~5 → 70점
  UV 6~7 → 40점
  UV 8 이상 → 10점

바람점수:
  0~4m/s → 100점
  5~8m/s → 70점
  9~13m/s → 40점 (주의)
  14m/s 이상 → 10점 (위험)

강수점수:
  강수확률 0~20% → 100점
  21~40% → 70점
  41~60% → 40점
  61% 이상 → 10점

코스보정 (코스 특성에 따른 가중치 조정):
  - 그늘 없음 (해안·능선) → UV점수 가중치 ↑
  - 급경사 구간 있음 → 바람점수 가중치 ↑
  - 산악 코스 → 강수점수 가중치 ↑
```

**5단계 등급**

| 등급 | WCI 점수 | 색상 | 설명 |
|---|---|---|---|
| 최적 | 85~100 | 초록 | 지금 바로 출발하세요 |
| 양호 | 70~84 | 연초록 | 가볍게 준비하고 출발 가능 |
| 보통 | 55~69 | 노랑 | 자외선 차단·보온 준비 권장 |
| 주의 | 35~54 | 주황 | 일정 재검토 또는 방비 철저 |
| 위험 | 34 이하 | 빨강 | 출발 자제 권고 |

### F4. 날씨 창(Weather Window) + 최적 출발 시점 추천

**개요**: 코스 소요시간과 날씨 예보를 매칭해, 전 구간을 쾌적하게 걸을 수 있는 출발 시점을 최대 3개 추천한다.

**로직**

```
입력:
  - course_id
  - target_date        (여행 예정일)
  - preferred_time     (선호 시간대, optional)

탐색:
  for start_hour in 05:00 to 17:00 (30분 단위):
    simulate_walk(start_hour, course)
    if max_rain_prob < 40% AND max_wind < 10m/s AND wci > 70:
      candidate_windows.append(start_hour)

정렬:
  score = WCI_avg - rain_prob_max * 0.5
  top 3 windows by score

출력:
  - 추천 출발 시각 (1~3개)
  - 각 시각별 WCI, 날씨 요약, 주의사항
  - "오늘 이 코스의 최적 출발은 오전 7시입니다" 문구
```

**알림 연동**

- 사용자가 추천 시각 선택 → "출발 30분 전 알림" 설정
- 날씨 급변 시 "출발 예정 2시간 전, 비 예보 추가됨" 긴급 알림

### F5. 코스 저장 + 날씨 모니터링

**개요**: 저장한 코스에 예정일을 지정하면, 날씨 예보 변동을 자동으로 모니터링하고 알림을 보낸다.

**기능 흐름**

```
코스 저장
  → 예정일 설정 (캘린더 피커)
  → 모니터링 ON

모니터링 스케줄:
  - D-7: 기상 예보 첫 확인, 초기 WCI 전달
  - D-3: 예보 갱신, 유의미한 변화 시 알림
  - D-1: 최종 날씨 확인 알림 (반드시 발송)
  - D-day 출발 2시간 전: 실시간 날씨 기반 최종 WCI + 출발 권고

알림 유형:
  - "OO 코스 D-3, 오전 날씨 좋습니다 ☀️ 최적 출발 시각: 오전 8시"
  - "OO 코스 D-1, 오후부터 비 예보로 변경됐습니다 ☂️ 오전 코스 추천"
  - "OO 코스 내일 WCI 위험 ⚠️ 날씨 변동 큽니다. 일정 재검토를 권장합니다"
```

**저장 제한**

| 티어 | 저장 코스 수 | 모니터링 코스 수 |
|---|---|---|
| 무료 | 최대 2개 | 1개 |
| 프리미엄 | 한도 확대 | 한도 확대 |

### F6. 위험 기상 경보 (Safety Alert)

**개요**: 기상청 특보를 코스 경로와 공간 매칭해, 해당 코스에 영향을 미치는 기상 특보를 즉시 알린다.

**연동 특보 유형**

| 특보 | 코스 유형 | 위험도 |
|---|---|---|
| 낙뢰 주의보·경보 | 산악·능선·해안 | 최고 |
| 강풍 주의보·경보 | 해안·능선·고지대 | 높음 |
| 호우 주의보·경보 | 전 코스 | 높음 |
| 대설 주의보·경보 | 산악 | 높음 |
| 폭염 특보 | 그늘 없는 코스 | 보통 |
| 한파 특보 | 겨울 고지대 | 보통 |
| 안개 주의보 | 해안·산악 | 보통 |

**매칭 로직**

```
for each active_alert in kma_alerts:
  if course_polygon intersects alert_region:
    severity = evaluate(alert.type, course.exposure_type)
    push_notification(user, course, alert, severity)
```

**알림 예시**

- "⚡ 북한산 둘레길 5코스 — 현재 낙뢰 주의보 발효 중. 즉시 하산 권고"
- "🌊 제주 올레 7코스 — 강풍 경보. 해안 구간 출입 위험"
- "🌧 지리산 둘레길 — 내일 호우 예보. 산사태 위험 코스 주의"

### F7. 내 경로 등록 (사용자 커스텀 코스)

**개요**: 내장 DB에 없는 경로도 사용자가 직접 등록해 동일한 날씨 분석을 받을 수 있다.

**등록 방법**

1. **GPX 파일 임포트**: 등산앱(트랭글, 스트라바 등)에서 내보낸 GPX 파일 업로드
2. **지도 직접 그리기**: 앱 내 지도 위에 경로 탭-탭 입력
3. **최근 걸은 경로 저장**: GPS 트래킹 후 "이 경로 저장"

**등록 후 제공 기능**

- 자동 거리·소요시간 계산 (평균 보행 속도 기준)
- F2~F6 전체 기능 동일 적용

**티어**: 프리미엄 전용

## 4. UX 흐름 (Screen Flow)

```
홈 화면
  └─ [도보여행] 탭 진입
       ├─ 오늘의 추천 코스 카드 (WCI 기준 상위 3개)
       ├─ 내 저장 코스 (예정일 D-day 카운트)
       └─ 코스 탐색 버튼

코스 탐색
  ├─ 필터: 지역 / 거리 / 난이도 / 오늘 날씨 좋음
  └─ 코스 목록 (WCI 배지 표시)

코스 상세
  ├─ 코스 기본 정보 (거리·시간·난이도)
  ├─ 오늘의 WCI + 등급
  ├─ 구간별 날씨 타임라인 (F2)
  ├─ 최적 출발 시각 추천 카드 (F4)
  ├─ 활성 특보 배너 (F6, 있을 경우)
  └─ [저장하기] 버튼

코스 저장 후
  ├─ 예정일 설정
  ├─ 알림 설정 (D-7, D-3, D-1, 당일)
  └─ 모니터링 ON → 홈 내 저장 코스 카드에 노출
```

## 6. 수익화 연계

### 무료 제공
- 코스 탐색 (전체 DB)
- 코스 상세 날씨 확인
- WCI 확인
- 최적 출발 시각 추천 (하루 3회)
- 저장 코스 최대 2개
- D-1 날씨 알림 1개

### 프리미엄 전용
- 코스 저장 한도 확대
- 날씨 모니터링 한도 확대 (D-7부터)
- 긴급 특보 알림
- GPX 임포트 / 커스텀 코스 등록
- 구간별 상세 날씨 (1시간 단위)

### 제휴 기회
- 아웃도어 브랜드 (블랙야크 · 코오롱스포츠 · 머렐): WCI 기반 "오늘 이런 신발 추천"
- 지역 관광청: 올레길·둘레길 공식 정보 배너
- 여행 플랫폼 (야놀자·여기어때): 코스 인근 숙소 연결
