## 6. Production Access 답변 초안

### 앱 테스트 방법

WeatherON은 Android preview APK를 통해 실제 기기와 에뮬레이터에서 홈, 온보딩, 위치 권한, 목적지 검색, 목적지 케어, 코디 추천, 정책 화면을 검증한다. 테스트 전 `npm run check:android-local-release-ready`로 스토어 자산, readiness 리포트, TypeScript, shared rule, mockup build, Expo config, Android release config를 통과시킨다.

### 테스터가 사용한 핵심 기능

- 현재 위치 기반 날씨 요약
- 날씨 기반 코디/우산/신발 추천
- 위치 권한 허용/거부/fallback
- Kakao Local 목적지 검색
- 목적지 저장 및 목적지 케어 상태
- 계정 연결 게이트와 복귀 흐름
- 개인정보/위치/광고 정책 화면

### 수집한 피드백 처리 방식

테스터 피드백은 화면, 심각도, 재현 조건, 조치 상태로 분류한다. High 이슈는 다음 APK 배포 전 수정하고, Medium/Low 이슈는 출시 전 수정 또는 알려진 제한사항으로 기록한다.

### 출시 준비 판단 기준

프로덕션 제출 전 필수 플로우가 크래시 없이 동작하고, 위치 권한 거부 상태에서도 앱 기본 기능이 유지되며, API 실패 시 빈 화면 대신 fallback 또는 재시도 상태가 표시되어야 한다.
