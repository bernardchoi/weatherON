기기 언어 자동 적용 → 스플래시 → Guest 홈 우선 진입 → 계정 필요 기능에서 로그인 요청 → 홈 / 출발 / MY MVP 기능 검증 플로우. 코디와 소셜(ON Square)은 확장 레이어로 유지한다. AUTH는 첫 관문이 아니라 저장·동기화·알림 확장·여행 플래너 등 계정 필요 기능의 보조 플로우로 동작한다. 로그인 수단은 한국/일본/일반 해외 3권역 기준으로 자동 추천하되, 현재 여행 위치보다 스토어 국가·SIM 국가·OS 지역·기기 언어 등 계정 환경 신호를 우선한다. 국가 선택을 가입 관문으로 만들지 않는다. 중국권 등 특수 인증·네트워크 환경 국가는 1차 범위에서 제외한다. PLACE·ADMOB은 출발·홈·MY·AUTH·온보딩에 부속되는 상세 확장 화면. 구조·정보 위계는 유지하고, WeatherON CI/BI의 Navy, Sky, Clear, Gold 토큰을 적용한 미드파이 단계. 시각 기준은 docs/design/WeatherON_UI_Design_Spec.md를 따르며, 목업과 와이어프레임이 다르면 목업 기준을 우선한다. MVP 기능 개발 기준은 docs/planning/WeatherON_MVP_기능_PRD.md, 기술 구현 기준은 docs/architecture/WeatherON_MVP_기술_ADR.md를 따른다.

목업 프리뷰 상태 기준: 로그인·권한·위치·프리미엄처럼 앱 전역에 남아야 하는 값은 Session State(accountLinked/permissionReady/locationReady/premiumActive)로 유지하고, 스타일 기준·핏·연령대·스마트 알림 세부 토글처럼 설정에 남아야 하는 값은 Settings State(styleProfileSaved/selectedStyles/fit/age/smartCareEnabled/alertAdvancedOpen/rainDetail/routineAlert/bedtimeAlert/destinationAlert/quietHours)로 유지한다. 선택 목적지·코디 조합·복귀 액션처럼 특정 흐름에 필요한 값은 Route State로 전달한다. Session/Settings State 키는 전역값을 우선 적용해 오래된 화면 컨텍스트가 로그인·권한·구독·설정 완료 상태를 되돌리지 않게 한다. 따라서 A2/A3, O3, G6, O4, H2, M2를 거친 뒤 사이드바나 하단 탭으로 다른 화면에 이동해도 전역/설정 상태가 끊기지 않아야 한다.
