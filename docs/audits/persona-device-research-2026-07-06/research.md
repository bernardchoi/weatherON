# WeatherON persona device research

날짜: 2026-07-06

## 연구 범위

- 제품: WeatherON Android MVP
- 대상: 마케팅 문서 기준 `출근/등교/약속 전 5분 안에 외출 준비를 판단해야 하는 사용자`
- 기간: MVP1 비공개 테스트 직전 관점
- 범위: 온보딩, 홈, 목적지/출발, 알림, 권한, MY
- 현재 상태: 2026-07-06 ADB 연결 기기 없음. 새 실기기 캡처는 못 함.
- 실기기 근거: 2026-07-04, 2026-07-05 Android real-device QA 산출물 사용.
- 보조 근거: 2026-07-06 web export persona audit 사용. 단, web 근거는 실기기 근거와 분리함.

## 핵심 요약

WeatherON의 제품 방향은 마케팅 타겟과 맞음. 홈은 날씨 숫자보다 나갈 시간, 비 그침, 챙길 것을 먼저 보여주며, 실기기 최종 QA에서도 홈/알림/MY 핵심 화면은 크래시 없이 동작 확인됨. 다만 실사용 관점에서는 "목적지 없음" 상태에서 사용자가 바로 얻는 가치가 아직 약하고, 목적지 검색은 자동 입력 검증이 막혀 실제 손입력 재검증이 필요함. 알림은 권한 전 예시와 권한 복구 흐름이 정리됐지만, 알림 센터/설정에서 권한, 예시, 조건 설정의 관계는 초반 사용자에게 여전히 설명 부담이 있음. MY는 구조가 명확하지만, 계정 연결 전 상태에서 권한/알림/목적지 저장의 선후관계를 더 단순하게 보여줄 여지가 있음. 공개 사용자 리뷰는 앱 비공개 테스트 전 상태라 신뢰 가능한 외부 불만 신호가 없음.

## 마케팅 타겟 기반 페르소나 10명

1. 출근 직장인
   - 목표: 출근 전 5초 안에 나갈 시간, 우산, 겉옷 판단.
   - 실기기 관찰: 홈에서 현재 날씨, 다음 비, 목적지 없음, 나갈 시간 카드 확인됨.
   - 피드백: 목적지 저장 전에는 `나갈 시간`이 실제 출근 판단보다 `목적지 추가` 유도로 읽힐 수 있음.

2. 등교 대학생
   - 목표: 수업 장소 기준 비/출발 시간 확인.
   - 실기기 관찰: 목적지 검색 진입은 확인됐으나 ADB 입력으로 실제 결과 검증은 불가했음.
   - 피드백: 실사용 검색어 `서울역`, `강남역`, 학교명 손입력 검증이 먼저 필요함. 앱 버그로 단정하지 않음.

3. 약속 전 사용자
   - 목표: 약속 장소에 맞춰 언제 나가야 하는지 확인.
   - 실기기 관찰: 목적지 케어 화면에서 출발/도착 계산과 시간 선택 UI가 확인됨.
   - 피드백: 시간 선택 UI가 판단 정보보다 강하게 보일 수 있음. 기본 계산 결과 우선, 수정은 접힘이 적합함.

4. 외근 많은 직장인
   - 목표: 여러 목적지 중 오늘 갈 곳을 빠르게 고름.
   - 실기기 관찰: 출발 탭과 목적지 없음 상태는 동작 확인됨.
   - 피드백: 여러 목적지 정렬, 최근 목적지, 오늘 목적지 우선순위는 현재 실기기 근거로 확인 못 함. MVP1 테스트 항목으로 필요함.

5. 비 예민 사용자
   - 목표: 비 시작/그침, 우산 필요 여부 판단.
   - 실기기 관찰: 홈에서 다음 비, 강수, 우산 카드가 보임.
   - 피드백: 시간 표기만으로 오늘/내일 맥락은 약할 수 있음. `오늘 21:00`, `내일 04:00` 같은 날짜 맥락 필요함.

6. 위치 권한 거부 사용자
   - 목표: 위치 권한 없이 수동 위치로 앱 사용.
   - 실기기 관찰: MY/권한 관리에서 수동 위치와 권한 복구 경로 확인됨.
   - 피드백: 홈의 `수동`, `기본 위치 서울` 영역이 위치 변경 진입점인지 더 명확해야 함.

7. 알림 거부 성향 사용자
   - 목표: 알림 없이도 목적지 비교 사용.
   - 실기기 관찰: 권한 전 예시, 권한 관리, 스마트 알림 설정 확인됨.
   - 피드백: 알림 없이 사용하는 경우에도 "앱 내에서 무엇까지 가능한지"를 더 직접적으로 알려야 함.

8. 알림 의존 사용자
   - 목표: 비/출발 알림을 믿고 앱을 닫음.
   - 실기기 관찰: 알림 권한 허용 후 Android notification manager 등록이 확인된 기록 있음.
   - 피드백: 실제 수신 확인 완료 피드백은 중요함. 기존 기록에서 예약 후 화면 피드백 부족이 지적됐고, 이후 문구 수정 반영됨. 재실기기 확인 필요함.

9. 주말 도보여행 사용자
   - 목표: 코스 이동 중 시간대별 날씨 확인.
   - 실기기 관찰: 현재 MVP 실기기 근거는 홈/출발/목적지/알림 중심임. 코스/구간별 날씨는 확인 범위 아님.
   - 피드백: MVP에서는 도보여행을 전면 기능으로 보지 말고, 목적지 케어 안의 "걷는 시간대 날씨" 수준으로 수요 검증 필요함.

10. 접근성 민감 사용자
   - 목표: 작은 글씨, 색 대비, 스크린리더 탐색 부담 최소화.
   - 실기기 관찰: 홈 알림 버튼 접근성 라벨은 정상 확인됨. UI tree 덤프도 확보됨.
   - 피드백: 시간 선택 휠, 하단 CTA, 작은 보조 텍스트는 TalkBack/큰 글자 설정 재검증 필요함. WCAG 준수는 이번 근거만으로 단정 불가함.

## 랭킹된 UX 문제

### 1. 목적지 없는 첫 홈의 즉시 가치가 약함

- 사용자 목표: 앱을 열자마자 오늘 준비 판단.
- 표면: 홈 H1.
- 관찰: 실기기 최종 홈은 날씨와 준비 카드가 잘 보이나, 목적지 없음 상태에서 `나갈 시간`은 목적지 추가 의존으로 보임.
- 무엇이 깨짐: 앱 가치가 "바로 판단"인데, 일부 핵심 판단이 설정 전 상태로 보임.
- 근거: `docs/audits/android-real-device-uiux-2026-07-05/screens/19-final-home.png`
- 심각도: Medium.
- 빈도 신호: 핵심 첫 실행 사용자 전원에게 노출 가능.
- 확신도: High.
- 권장: 목적지 미등록 시에도 "현재 위치 기준 바로 쓸 수 있는 판단"과 "목적지 추가 시 강화되는 판단"을 나눠 표시.

### 2. 목적지 검색 실사용 검증이 부족함

- 사용자 목표: 실제 장소를 찾아 출발/알림에 연결.
- 표면: 목적지 추가/검색.
- 관찰: 실기기 QA에서 ADB 입력이 한국어 IME 조합 문자열로 들어가 검색 결과 품질 검증이 불가했음.
- 무엇이 깨짐: 앱 버그는 아님. 다만 MVP1 핵심 전제인 검색-선택-저장 신뢰를 아직 실사용 방식으로 증명 못 함.
- 근거: `docs/audits/android-real-device-uiux-2026-07-05/report.md`, `screens/06-place-search-seoul.png`, `ui/06-place-search-seoul.xml`
- 심각도: High.
- 빈도 신호: 목적지 기능을 쓰는 사용자 대부분에게 영향.
- 확신도: Medium. 문제 자체는 검증 공백이며, 앱 결함 확정 아님.
- 권장: 실기기 손입력으로 `서울역`, `강남역`, `인천공항`, 학교/회사명 검색-선택-저장-재시작 유지까지 확인.

### 3. 알림 권한 전 상태의 정보 구조가 여전히 무거움

- 사용자 목표: 알림을 켜야 하는 이유와 현재 가능한 기능 이해.
- 표면: 알림 센터, 스마트 알림 설정, MY.
- 관찰: 최종 알림 센터는 `권한 전 예시`, `조건 설정`, `전체 읽음`, 예시 알림 카드가 함께 보임.
- 무엇이 깨짐: 권한을 켜기 전 사용자는 예시/실제/조건 설정의 차이를 배워야 함.
- 근거: `docs/audits/android-real-device-uiux-2026-07-05/screens/21-final-notification-center.png`
- 심각도: Medium.
- 빈도 신호: 신규 사용자와 알림 미허용 사용자에게 반복 노출.
- 확신도: Medium.
- 권장: 권한 전에는 "예시 보기"와 "권한 켜기"만 남기고 조건 설정은 권한 후 또는 고급으로 이동.

### 4. 권한/계정/저장 gate의 선후관계가 더 단순해야 함

- 사용자 목표: 가입 전 둘러보고, 필요한 순간만 연결.
- 표면: MY, 권한 관리, 목적지 저장/알림.
- 관찰: MY에는 게스트 모드, 계정 연결, 권한 관리, 스마트 알림 설정이 분리되어 보임. 구조는 정상이나 처음 쓰는 사용자에게 선후관계 설명이 필요함.
- 무엇이 깨짐: "계정 연결 후 저장 가능", "권한은 알림용", "수동 위치는 권한 없이 가능"의 차이가 한눈에 완전히 정리되지는 않음.
- 근거: `docs/audits/android-real-device-uiux-2026-07-05/screens/22-final-my.png`
- 심각도: Medium.
- 빈도 신호: 게스트 사용자, 권한 거부 사용자.
- 확신도: Medium.
- 권장: MY 상단에 `지금 가능 / 계정 필요 / 권한 필요` 3단 상태 요약을 추가.

### 5. 목적지 케어의 시간 조정 UI가 판단보다 앞설 위험 있음

- 사용자 목표: 몇 시에 나가야 하는지 빠르게 확인.
- 표면: 목적지 케어 G2.
- 관찰: 2026-07-04 실기기 캡처에서 목적지 케어 화면은 작동하나 nested frame과 시간 선택 영역이 무겁다고 기록됨.
- 무엇이 깨짐: 계산 결과 확인보다 설정 조작이 먼저 보일 수 있음.
- 근거: `docs/audits/2026-07-04-product-design-uiux/01-destination-care.png`, `docs/audits/2026-07-04-product-design-uiux/audit.md`
- 심각도: Medium.
- 빈도 신호: 목적지 저장 사용자.
- 확신도: Medium. 이후 일부 개선 가능성이 있어 재실기기 확인 필요.
- 권장: 계산 결과를 상단 고정하고, 시간 수정은 `도착 시간 바꾸기` 접힘 컨트롤로 축소.

### 6. 시간 표현에 날짜 맥락이 부족할 수 있음

- 사용자 목표: 비 시작/그침을 실제 일정에 맞춰 판단.
- 표면: 홈, 강수 카드.
- 관찰: 홈은 `다음 비 21:00`처럼 시간 중심으로 보여줌.
- 무엇이 깨짐: 밤/새벽 시간대는 오늘/내일 혼동 가능.
- 근거: `docs/audits/android-real-device-uiux-2026-07-05/screens/19-final-home.png`
- 심각도: Low-Medium.
- 빈도 신호: 강수 예보가 자정을 넘는 날 반복 가능.
- 확신도: Medium.
- 권장: `오늘`, `내일`, `n시간 후`를 함께 표기.

## 관찰 증거와 추론 분리

### 관찰된 것

- 2026-07-05 실기기에서 핵심 UI/UX 점검 통과.
- 실기기 cold launch 성공, crash buffer 0 byte, 주요 JS/Android 오류 패턴 없음.
- 홈, 알림 패널, 알림 센터, MY 최종 캡처 있음.
- 알림 권한 흐름은 앱 내 설명 후 OS 권한 프롬프트로 이어진 기록 있음.
- 목적지 검색은 자동 입력 문제로 실사용 검색 결과 품질 검증이 부족함.

### 추론한 것

- 목적지 없음 홈은 실제 사용자가 가치 이해 전에 설정 과제를 먼저 느낄 수 있음.
- 알림 권한 전 예시/조건 설정은 신규 사용자에게 학습 부담이 될 수 있음.
- 시간 표현은 날이 넘어가는 상황에서 혼동될 수 있음.
- 도보여행 페르소나는 현재 MVP 직접 대상보다 확장 검증 대상에 가까움.

## 소스 맵

- 공개 소스:
  - WeatherON 자체 공개 리뷰/불만 신호는 확인 가능한 강한 소스 없음.
  - Android 공식 알림 권한 문서: Android 13+ 신규 설치 앱은 알림 권한이 기본 off이고, 앱은 권한 요청 전 사용자에게 이유를 설명할 수 있음. `https://developer.android.com/develop/ui/compose/notifications/notification-permission`
  - Android 공식 vitals 문서: cold startup 5초 이상은 excessive 기준. WeatherON 실기기 cold launch 기록은 이 기준보다 낮았음. `https://developer.android.com/topic/performance/vitals/launch-time`
- 내부 소스:
  - `docs/intro/WeatherON_소개서_v1.html`: 마케팅 타겟, 핵심 가치.
  - `docs/planning/WeatherON_MVP_기능_PRD.md`: MVP 목표, 권한/계정/알림 구조.
  - `docs/audits/android-real-device-uiux-2026-07-05/report.md`: 최종 실기기 QA, 로그, 화면 근거.
  - `docs/audits/product-design-full-ux-2026-07-05/audit-notes.md`: 실기기 전체 UX/알림/권한 근거.
  - `docs/audits/2026-07-04-product-design-uiux/audit.md`: 목적지 케어/홈/출발/MY 실기기 디자인 근거.
  - `docs/audits/persona-user-review-2026-07-06/audit.md`: web export 보조 근거. 실기기 근거로 취급하지 않음.

## 기회 맵

### 이번 주

1. 실기기 손입력 목적지 검색 QA 추가.
2. 홈 목적지 없음 상태에서 "바로 판단 가능"과 "목적지 추가 필요" 분리.
3. 알림 권한 전 센터에서 조건 설정 CTA 약화.
4. 비 시간에 오늘/내일 맥락 추가.

### 이번 분기

1. 목적지 2개 이상 저장 시 정렬/우선순위 UX 검증.
2. 알림 권한 거부 사용자용 제한 없는 기능 요약 추가.
3. TalkBack, 큰 글자, 작은 화면 실기기 QA 루틴화.
4. 목적지 케어 시간 선택 UI를 접힘 편집 패턴으로 정리.

### 추가 리서치 필요

1. 10~30명 MVP1 테스터가 목적지 추가 없이도 홈 가치를 이해하는지.
2. 알림 권한 전 예시가 안심을 주는지, 가짜 알림처럼 느껴지는지.
3. 비 그침/출발 알림이 실제 외출 판단에 쓰이는지.
4. 도보여행 사용자가 MVP 홈/목적지 케어만으로 충분한 예비 가치를 느끼는지.

## 허구 방지 메모

- 현재 ADB 연결 기기 없음. 새 실기기 문제는 추가 주장하지 않음.
- 목적지 검색은 앱 버그로 단정하지 않음. 실기기 자동 입력 방식 한계로 기록함.
- 2026-07-05 수정 후 해소된 `WeatherON Dev`, 신규 알림 배지 정책, 알림 수신 피드백 문제는 현재 문제로 재등재하지 않음.
- 실제 푸시 장기 신뢰성, GPS 정확도, 목적지 다중 저장 UX는 이번 근거로 확정하지 않음.
