## 주요 문제

1. 시작 화면 CTA가 첫 화면 밖에 있음
   - `나가기 전 5초 판단`이 핵심인데 실제 첫 화면은 설명과 카드가 먼저 차지함.
   - 사용자가 바로 홈으로 들어가려면 스크롤 필요함.
   - 수정 방향: 핵심 3개 수치와 `홈에서 먼저 판단` CTA를 첫 viewport 안에 고정 배치 필요.

2. 홈 화면이 MVP 핵심 판단을 과하게 많이 보여줌
   - 출발 시간, 목적지 비교, 강수 타임라인, 위치, 목적지 케어, 코디, 우산, 알림 센터가 한 흐름에 이어짐.
   - “오늘 바로 결정” 섹션까지는 좋으나 이후 보조 기능이 길게 붙어 5초 판단성이 약해짐.
   - 수정 방향: 첫 화면은 `언제 나감`, `목적지는 다른가`, `비는 언제 그침`만 남기고 코디/우산/알림은 접힌 추천 영역으로 축소 필요.

3. 홈 CTA 연결 구분 약함
   - `출발 준비 보기`와 `목적지 비교`가 같은 목적지 기준 화면으로 이동함.
   - CTA 이름은 다른데 도착 화면이 같아 사용자는 “내가 맞게 눌렀나” 헷갈릴 수 있음.
   - 수정 방향: 하나로 합치거나, 도착 화면의 앵커/상단 제목을 CTA별로 다르게 맞춰야 함.

4. 코디 기준 수정 화면이 흐름을 가둠
   - `기준 수정` 진입 후 하단 탭 없음.
   - `MY에 저장`, `코디로`만 보여 다른 주요 탭 이동이 막힘.
   - 이후 좌표 입력이 스타일 선택으로 처리될 만큼 화면 탈출성이 약함.
   - 수정 방향: 상단 뒤로가기와 하단 탭 유지, 또는 명확한 닫기 버튼 필요.

5. MY 화면 접근성 이름이 길고 중복됨
   - 버튼 aria가 대부분 `null`.
   - 카드 전체 텍스트가 버튼명으로 잡혀 `알림 설정 권한 확인 필요 스마트 알림 관리 확인`처럼 길어짐.
   - 수정 방향: 카드형 버튼에 짧은 `accessibilityLabel` 부여 필요. 예: `알림 설정 열기`, `전역 설정 열기`.

6. 상태바/앱 프레임 일관성 깨짐
   - 출발/MY는 `9:41`, `5G` 상태바가 보임.
   - 홈/코디/소셜 일부는 브랜드 헤더 중심으로 보임.
   - 같은 앱 안에서 네이티브 프레임과 목업 프레임이 섞인 느낌 있음.
   - 수정 방향: 실제 앱 기준 상태바 처리 규칙 하나로 통일 필요.

7. 개인정보/정책 허브 복귀 흐름 약함
   - 정책 화면은 하단 탭이 사라지고 문서형 목록만 남음.
   - 설정 허브에서 들어온 뒤 돌아갈 위치가 명확하지 않음.
   - 수정 방향: 상단 뒤로가기, 설정으로 돌아가기, 하단 탭 중 하나는 유지 필요.

8. 소셜/ON Square는 MVP 핵심과 연결 약함
   - 화면 자체는 연결되나 홈의 5초 판단, 출발 알림, 목적지 날씨와 직접 이어지는 행동이 적음.
   - 수정 방향: MVP에서는 탭 노출 강도를 낮추거나 `날씨 노트`를 홈/강수 흐름의 보조 증거로 연결 필요.

9. CTA와 상태 문구가 기능 검증용 문장처럼 남아 있음
   - `MVP 0에서 검증할 핵심 흐름`, `MVP 이후`, `PROFILE STATE`, `선택 항목 정책 허브 대기` 등이 사용자 화면에 노출됨.
   - 수정 방향: 내부 검증 문구는 QA 빌드 전용 플래그로 숨기고 사용자 문장으로 교체 필요.

10. 색 대비와 작은 보조 텍스트 리스크 남음
    - 어두운 배경 위 저대비 회색/청색 보조 텍스트가 많음.
    - 일부 칩/보조 버튼은 터치 목표가 작아 보임.
    - 수정 방향: 보조 텍스트 대비 상향, 칩 최소 터치 영역 44px 기준 재점검 필요.
