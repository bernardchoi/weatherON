## 한줄 판정

앱처럼 보이기는 함. 다만 출시 앱이라기보다 고충실도 MVP 프리뷰에 가까움. 현재 체감 완성도는 목업 기준 약 55~60% 수준.

## 단계별 상태

| Step | 화면 | 상태 | 근거 |
|---:|---|---|---|
| 1 | 홈 | 보통 | 정보 구조는 앱 같지만 상단 날씨 블록과 카드가 무거움. 하단 3탭은 MVP 런치 표면 기준으로 확정됨 |
| 2 | 강수 타임라인 | 좋음 | 기능 목적이 명확하고 카드/차트/추천 흐름이 자연스러움 |
| 3 | 우산 추천 | 보통 | 추천 근거는 좋지만 CTA와 탭바가 충돌하고 비교 카드가 비활성처럼 보임 |
| 4 | MY | 좋음 | 설정 허브로 앱 느낌이 가장 안정적임. 메뉴/아이콘/상태 뱃지 구성이 정리됨 |

## 강점

- 홈, 강수, 우산, MY가 하나의 모바일 앱 흐름으로 연결됨.
- 강수 타임라인은 `요약 -> 알림 -> 차트 -> 가이드 -> 우산 추천` 순서가 자연스러움.
- MY는 실제 앱 설정 화면처럼 정보량과 터치 영역이 안정적임.
- 아이콘 애셋 기반 화면이 늘어나면서 텍스트 기호 느낌이 줄었음.
- 라이트모드에서 강수 차트와 토글 가시성은 이전보다 개선됨.

## UX 리스크

- 홈 상단이 너무 큼. 날씨 원형 영역, 칩 6개, 카드 3개가 한 화면에 몰려 첫 인상이 무거움.
- `기본 위치 서울`, `위치 수동`이 홈 상단에서 바로 보여 신뢰감이 약해짐. 사용자는 위치 권한 허용 후에도 앱이 자동으로 못 잡는다고 느낄 수 있음.
- 하단 내비는 MVP 런치 표면 기준 `홈/출발/MY` 3탭으로 확정됨. 코디/소셜 기능은 숨김 확장 라우트로 보존함.
- 강수 타임라인과 우산 추천에서도 하단 내비 active가 `홈`으로 보임. 현재 위치 인지가 약함.
- 우산 추천 하단 CTA가 탭바 바로 위에 붙어 강한 버튼끼리 경쟁함.
- 우산 종류 비교의 비추천 카드가 회색 비활성 UI처럼 보여 선택 불가/오류 상태로 오해 가능함.

## 접근성 리스크

- 홈/우산 화면 일부 텍스트가 굵고 크지만 카드 안 정보가 많아 읽기 피로가 큼.
- 우산 화면 고정 CTA와 탭바가 가까워 터치 실수 가능성이 있음.
- 하단 탭 active 표현이 큰 테두리 카드라 시각적으로 명확하지만, 목업의 가벼운 탭바보다 화면 공간을 많이 씀.
- 스크린샷만으로 TalkBack 순서, 동적 글자 크기, 실제 터치 타겟은 확정 판단 불가.

## 우선 개선

1. 홈 첫 화면 밀도 줄이기
   - 상단 칩 6개를 3개 이하로 축소하거나 일부를 접힘 정보로 이동.
   - 첫 화면에서 핵심 판단 카드 2개만 보이게 정리.

2. 위치 신뢰감 정리
   - 권한 허용 상태라면 `기본 위치 서울`보다 실제 주소/동 단위 표시 우선.
   - 수동 위치일 때는 사용자 행동 CTA를 분명히 제공.

3. 하단 내비 정책 유지
   - MVP 3탭 유지.
   - 목업/기획 문서는 3탭 기준으로 업데이트.

4. 서브 화면 active 상태 정리
   - H4/H5 같은 홈 하위 화면에서 하단 탭을 홈 active로 둘지, 하단 내비를 숨길지 정책 필요.
   - 현재처럼 홈 active 유지 시 사용자가 홈에 있는 것처럼 느낄 수 있음.

5. 우산 추천 CTA 안전영역 개선
   - 고정 CTA와 탭바 사이 간격 확대 또는 우산 화면에서는 하단 내비 숨김 검토.
   - 비교 카드는 비활성 회색 대신 중립 카드 톤으로 조정.

## 앱처럼 느껴지는 정도

- 화면 구성: 65%
- 브랜드 일관성: 60%
- MVP 문서 정합성: 65%
- 실제 앱 체감: 55~60%
- 출시 전 디자인 완성도: 아직 부족
