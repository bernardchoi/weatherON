# WeatherON 실기기 페르소나 리서치 리뷰

- 일시: 2026-07-06
- 기기: Android A142, 1084x2412, density override 375
- 앱: `com.weatheron.mobile`, WeatherON v0.1.0, versionCode 6
- 방식: 기존 앱 데이터 유지, Android 실기기에서 ADB 화면 캡처/UiAutomator 덤프
- 주의: 실제로 보인 화면만 근거로 삼음. 검색 입력은 ADB 텍스트 주입으로 한글 IME 조합이 깨져 검색 품질 문제로 단정하지 않음.

## 요약

WeatherON은 마케팅 문서의 핵심 약속인 `나갈 시간`, `비 시작/그침`, `챙길 것`, `목적지 케어`, `알림`을 실기기 화면에서 실제로 보여줌. 홈, 출발, 알림 센터, 강수 상세, 목적지 상세, 권한, 스마트 알림 설정까지 주요 연결은 끊기지 않음. 가장 큰 개선점은 기능 부재보다 “같은 판단 정보를 여러 표현으로 보여줘 사용자가 다시 해석해야 하는 부분”임. 특히 강수 시간 표현이 홈/출발/강수 상세 사이에서 다르게 읽히고, 출발 역산은 수식은 보이지만 `왜 20분 여유인지` 신뢰 근거가 약함. 권한과 알림 상태는 현재 허용 상태에서 잘 보이나, 비허용/첫 실행 상태는 앱 데이터 초기화 없이 검증하지 않았음. 목적지 검색은 ADB 입력 한계 때문에 이번 리포트에서 품질 판정을 보류함.

## 마케팅 타겟 기준 10명 페르소나

1. 출근 직장인: 아침 5분 안에 우산/출발 시간을 정해야 함.
2. 등교 대학생: 학교·알바 장소가 달라 목적지 날씨를 빠르게 비교해야 함.
3. 아이 등원 보호자: 아이 옷차림과 우산을 동시에 판단해야 함.
4. 외근 영업직: 이동수단과 도착 시각 기준으로 출발 알림이 필요함.
5. 자차 출퇴근자: 도로 기준 이동시간과 비 시작 시간을 같이 봐야 함.
6. 대중교통 이용자: 배차/환승 변동 가능성을 알고 싶음.
7. 야외 스포츠 관람객: 경기장 목적지의 강수와 신발/우산 준비가 중요함.
8. 교회·모임 방문자: 반복 일정 목적지 알림을 저장하고 싶음.
9. 주말 도보 여행자: 걷는 시간대 전체의 강수 변화를 보고 싶음.
10. 여행 중 관광객: 낯선 장소 검색과 수동 위치 선택이 중요함.

## 발견 이슈 및 개선점

### P1. 강수 시간 의미가 화면마다 다르게 읽힘

- 사용자 목표: 비가 언제 시작하고 언제 그치는지 5초 안에 판단.
- 영향 페르소나: 출근 직장인, 등교 대학생, 보호자, 도보 여행자.
- 관찰: 홈은 `다음 비 04:00`, 카드에는 `비 그침 20:00 / 04:00 시작`, 강수 상세는 `04:00 전후 비 가능`, `시작 04:00`, `그침 04:00`으로 보임.
- 문제: 실제 데이터 오류라고 단정할 수는 없으나, 사용자가 “04:00에 시작인지, 20:00에 그치는지, 04:00에 그치는지”를 다시 해석해야 함.
- 근거: `01-home.png`, `03-notification-detail.png`, `04-departure.png`.
- 신뢰도: 높음. 화면에 직접 표시됨.
- 개선: 홈/알림/강수 상세의 라벨 체계를 통일 필요. 예: `다음 비 시작`, `비 약해지는 시간`, `외출 부담 낮아지는 시간`처럼 의미를 분리.

### P1. 출발 역산은 보이나 신뢰 근거가 부족함

- 사용자 목표: 도착 10:00 기준으로 09:09 출발을 믿고 따를 수 있어야 함.
- 영향 페르소나: 외근 영업직, 자차 출퇴근자, 대중교통 이용자, 모임 방문자.
- 관찰: 목적지 상세에 `10:00 - 이동 31분 - 여유 20분`, `Kakao Directions · 12.5km`가 표시됨.
- 문제: 수식은 좋으나 `자동 여유 20분` 산정 이유가 화면 안에서 설명되지 않음. 교통/날씨/사용자 패턴 중 무엇 때문인지 불명확함.
- 근거: `05-destination-detail.png`.
- 신뢰도: 높음. 화면에 직접 표시됨.
- 개선: `여유 20분`을 탭하면 산정 이유를 짧게 표시 필요. 예: `비 예보 + 현재시각 기준 기본 여유`.

### P1. 목적지 케어 핵심 카드가 정보는 많으나 우선순위가 흐림

- 사용자 목표: 목적지별로 지금 뭘 해야 하는지 바로 결정.
- 영향 페르소나: 야외 스포츠 관람객, 모임 방문자, 외근 사용자.
- 관찰: 출발 탭 목적지 카드에 장소, 지역, 기온 변화, 출발/도착, 알림 상태, 강수 80%, 반복 요일, 업데이트 상태가 한 카드에 밀집됨.
- 문제: `알림 켬`, `80%`, `방금 업데이트됨`, `반복 일` 중 무엇을 먼저 봐야 하는지 약함.
- 근거: `04-departure.png`.
- 신뢰도: 중간. 기능 오류가 아니라 정보 위계 문제임.
- 개선: 카드 1순위는 `출발까지 남은 행동`으로 두고, 강수/반복/업데이트는 보조 상태로 낮춤.

### P2. 홈의 현재 위치 텍스트가 잘려 위치 확신이 약함

- 사용자 목표: 내 현재 위치와 목적지 기준이 맞는지 즉시 확인.
- 영향 페르소나: 출근 직장인, 보호자, 여행자.
- 관찰: 홈 날씨 카드 위치가 `고양시 덕양구 삼송동 · 현재 ...`처럼 말줄임 처리됨.
- 문제: 날씨 판단의 기준 위치가 중요하지만 긴 주소에서 핵심 구분이 잘릴 수 있음.
- 근거: `01-home.png`.
- 신뢰도: 높음.
- 개선: 현재 위치는 1줄 요약 + 보조 줄 또는 `현재 위치` 칩 탭 시 전체 주소 표시.

### P2. 알림 설정의 “확인 알림” 상태와 보내기 버튼 의미가 모호함

- 사용자 목표: 실제 알림이 오는지 확인하고 안심.
- 영향 페르소나: 알림 의존도가 높은 외근 사용자, 보호자, 모임 방문자.
- 관찰: 스마트 알림 설정에 `수신 확인 전 · 5초 뒤 확인 알림 발송`과 `보내기` 버튼이 보이나, 버튼 활성/비활성 상태와 다음 행동이 명확하지 않음.
- 문제: 사용자는 지금 누르면 테스트 알림이 가는지, 이미 예약되어 기다려야 하는지 헷갈릴 수 있음.
- 근거: `09-notification-settings.png`.
- 신뢰도: 중간. 실제 발송은 누르지 않아 미검증.
- 개선: `테스트 알림 보내기`, `5초 뒤 도착 확인 중`, `다시 보내기 가능`처럼 상태와 동작을 분리.

### P2. 이동수단 변경 옵션은 보이나 선택 결과 비교가 약함

- 사용자 목표: 자차/도보/대중교통 중 무엇이 현실적인지 비교.
- 영향 페르소나: 대중교통 이용자, 자차 출퇴근자, 도보 여행자.
- 관찰: 이동수단 변경에서 자동/도보/자차/대중교통 옵션이 노출됨.
- 문제: 옵션별 예상 시간이나 주의점은 선택 전 화면에서 보이지 않음. 대중교통은 배차/환승 변동 가능 문구가 있지만 비교 판단은 약함.
- 근거: `06-transport-change.png`, PRD상 대중교통 변동 안내 요구 존재.
- 신뢰도: 중간. 선택 변경은 사용자 저장 상태 변경을 피하려고 수행하지 않음.
- 개선: 각 옵션에 예상 이동시간 또는 “선택 후 재계산” 안내 필요.

### P3. 목적지 검색 품질은 이번 검증에서 판정 보류

- 사용자 목표: 낯선 목적지를 빠르게 찾아 저장.
- 영향 페르소나: 여행자, 외근 사용자, 등교/약속 사용자.
- 관찰: ADB로 `Seoul` 입력 시 기기 한글 IME가 조합 문자열로 입력되어 0건이 표시됨.
- 문제: 이것은 앱 검색 버그로 확정할 수 없음. 입력 방식이 실제 손입력과 다름.
- 근거: `11-destination-search-seoul.png`.
- 신뢰도: 낮음.
- 개선: 별도 손입력 테스트 또는 테스트용 입력 API/디버그 빌드로 재검증 필요.

## 기회 맵

1. 강수 의미 체계 통일: 시작/최대/약해짐/그침/외출 부담 낮음 라벨을 전 화면 공통으로 정리.
2. 출발 역산 신뢰 보강: 이동시간, 여유시간, 날씨 영향, 교통 출처를 짧게 열람 가능하게 함.
3. 홈 5초 판단 강화: `지금 해야 할 1개 행동`을 최상단 카드로 압축.
4. 목적지 카드 위계 정리: 장소별 핵심 경고와 다음 행동을 우선 표시.
5. 알림 테스트 UX 명확화: 예약/발송/수신확인 상태와 버튼 동작을 분리.
6. 검색 재검증: 실제 손입력 또는 테스트 키보드 기준으로 한국어/영어/주소/랜드마크 검색을 따로 검증.

## 보류 항목

- 첫 실행 온보딩/게스트 흐름: 앱 데이터 초기화 없이 진행했으므로 미검증.
- 권한 거부 상태: 현재 기기는 위치/알림 허용 상태라 거부 복구 UX 미검증.
- 실제 푸시 도착 신뢰성: 테스트 알림 발송 버튼을 누르지 않아 미검증.
- 목적지 검색 품질: ADB 입력 한계로 미검증.
- 도보여행 전용 기능: 현재 MVP 화면에서는 독립 도보여행 흐름을 확인하지 못함.
