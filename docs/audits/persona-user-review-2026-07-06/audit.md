## 마케팅 타겟 해석

마케팅/기획 문서 기준 핵심 타겟은 "출근, 등교, 약속 전 5분 안에 준비 판단이 필요한 사람"임.
확장 타겟은 목적지 날씨 비교가 필요한 반복 이동자, 비 시작/그침 알림을 원하는 사용자, 주말 도보여행/트레킹 사용자임.

## 캡처 단계

1. 첫 홈 진입: 건강도 양호. 날씨, 목적지 없음, 나갈 시간, 비 그침, 챙길 것 노출됨. 근거: `screenshots/01-first-open.png`
2. 목적지 추가 진입: 건강도 보통. 검색 전 안내는 명확하나 하단 CTA는 비활성 상태임. 근거: `screenshots/02-destination-add.png`
3. 목적지 검색 결과: 건강도 양호. `강남역` 입력 후 결과 목록 노출됨. 근거: `screenshots/03-destination-search-results.png`
4. 목적지 선택: 건강도 양호. 선택 상태와 저장 CTA 활성화 확인됨. 근거: `screenshots/04-destination-selected.png`
5. 목적지 케어 상단: 건강도 보통. 출발 계산값은 보이나 시간 선택 UI가 화면 대부분을 차지함. 근거: `screenshots/05-destination-care.png`
6. 목적지 케어 하단: 건강도 보통. 알림 권한 CTA, 비교 상세, 삭제가 확인됨. 근거: `screenshots/06-destination-care-lower.png`
7. 알림 권한 진입: 건강도 양호. 권한 허용/알림 없이 저장 선택지가 분리됨. 근거: `screenshots/07-notification-permission-entry.png`
8. 알림 없이 저장: 건강도 보통. 저장 뒤 목적지 케어로 돌아옴. 근거: `screenshots/08-saved-without-notification.png`
9. 출발 탭 저장 목록: 건강도 양호. 저장 목적지, 권한 필요, 출발/도착 시간이 목록에 반영됨. 근거: `screenshots/09-depart-tab-saved.png`
10. MY 화면: 건강도 양호. 계정, 권한, 알림, 표시, 정책 진입 구조가 보임. 근거: `screenshots/10-my.png`
11. 앱 권한 관리: 건강도 양호. 위치/알림 권한 복구와 수동 선택 경로가 보임. 근거: `screenshots/11-permissions.png`
12. 스마트 알림 설정: 건강도 보통. 권한 상태와 알림 종류가 보이나 정보 밀도가 높음. 근거: `screenshots/12-alert-settings.png`
13. 홈 저장 반영: 건강도 양호. 저장 목적지와 나갈 시간/비 그침 카드가 홈에 반영됨. 근거: `screenshots/13-home-after-save.png`

## 10명 페르소나 리뷰

### 1. 출근 직장인

- 상황: 아침에 5초 안에 나갈 시간, 우산, 겉옷 판단 필요.
- 잘 맞는 점: 홈에서 `25도`, 강수, 다음 비, 나갈 시간, 비 그침이 한 화면에 나옴.
- 문제/개선: 첫 홈에서 목적지 없음 상태가 크고, 저장 전에는 `나갈 시간 09:30 목적지 추가`처럼 실제 판단보다 설정 유도가 섞여 보임.
- 개선안: 목적지 없는 첫 홈에는 "지금은 기본 위치 기준"과 "목적지 추가 시 바뀌는 값"을 더 짧게 분리 필요.

### 2. 등교 대학생

- 상황: 수업 시작 전 대중교통 이동, 비 예보 확인 필요.
- 잘 맞는 점: 목적지 검색 결과에서 역/장소 선택 가능함.
- 문제/개선: 검색 결과 중 `강남역센트럴푸르지오시티오피스텔`이 `등산` 카테고리로 보임. 버그라고 단정하지는 않지만, 사용자 입장에서는 카테고리 신뢰가 흔들릴 수 있음.
- 개선안: 장소 카테고리 표시를 숨기거나, 검색 제공처/주소를 우선 노출하는 편이 안전함.

### 3. 외근 많은 영업직

- 상황: 하루에 여러 목적지를 바꿔 확인함.
- 잘 맞는 점: 출발 탭에서 저장 목적지와 출발/도착 시간이 요약됨.
- 문제/개선: 현재 캡처에서는 저장 목적지가 1개일 때는 명확하지만, 여러 목적지 우선순위/정렬 기준은 확인 불가함.
- 개선안: 최근 확인, 오늘 일정, 권한 필요 같은 정렬 기준 표시 필요.

### 4. 약속 전 준비 사용자

- 상황: 약속 장소에 언제 나가야 하는지 빠르게 알고 싶음.
- 잘 맞는 점: 목적지 저장 후 `09:23 출발`, `10:05 도착` 계산이 바로 보임.
- 문제/개선: 도착 시간 선택 UI가 큼. 사용자는 "지금 몇 시에 나가야 해?"를 먼저 보려는데, 시간 조정 장치가 판단 정보보다 강하게 보임.
- 개선안: 기본 계산 결과를 먼저 고정하고, 시간 선택은 접기/수정 모드로 낮추는 편이 좋음.

### 5. 비 오는 날 우산 판단 사용자

- 상황: 비 시작/그침 시간과 우산 필요도를 보고 싶음.
- 잘 맞는 점: 홈에 `다음 비 04:00`, `비 그침 20:00`, 강수 카드가 보임.
- 문제/개선: 비 카드의 `04:00`, `20:00`이 오늘/내일 기준인지 화면만으로는 즉시 알기 어려움.
- 개선안: `오늘 20:00`, `내일 04:00`처럼 날짜 맥락 보강 필요.

### 6. 위치 권한을 꺼두는 사용자

- 상황: 위치 권한 없이 수동 위치로 쓰고 싶음.
- 잘 맞는 점: MY 권한 관리에 `수동 위치 기준`, `위치 선택` 경로가 있음.
- 문제/개선: 첫 홈에서는 `기본 위치 서울`로 동작하지만, 수동 위치 변경 경로가 홈 상단에서 즉시 드러나지는 않음.
- 개선안: 홈의 `수동` 배지 또는 `기본 위치 서울` 텍스트를 탭 가능한 위치 변경 진입으로 더 명확히 보여줄 필요 있음.

### 7. 알림 거부 성향 사용자

- 상황: 알림 권한은 거부하지만 목적지 저장은 하고 싶음.
- 잘 맞는 점: `알림 없이 저장` 경로가 존재하고 실제로 계속 진행됨.
- 문제/개선: 저장 뒤 다시 케어 화면으로 돌아와서 저장 완료 피드백이 약함. 출발 탭에서야 저장 반영이 확실히 확인됨.
- 개선안: 저장 직후 짧은 완료 상태나 `출발 탭에 저장됨` 안내 필요.

### 8. 알림 의존 사용자

- 상황: 비/출발 알림이 앱의 핵심 가치임.
- 잘 맞는 점: 스마트 알림 설정에서 필수 날씨, 생활 루틴, 목적지 출발이 구분됨.
- 문제/개선: `권한 필요`, `푸시만 대기`, `예약 0건`, `확인` 등 상태 문구가 많아 다음 행동이 흐려짐.
- 개선안: 권한 미허용 상태에서는 세부 알림 목록보다 `권한 켜기 -> 테스트 알림 확인 -> 목적지 알림 예약` 순서로 단계화 필요.

### 9. 주말 도보여행 사용자

- 상황: 목적지 하나보다 코스와 시간대별 날씨가 중요함.
- 잘 맞는 점: 현재 MVP 흐름은 목적지 날씨 비교와 출발 시간 계산의 기반은 있음.
- 문제/개선: 이번 캡처 기준으로 코스/구간별 날씨는 노출되지 않음. 도보여행 PRD 타겟에는 아직 직접 대응 부족함.
- 개선안: MVP에서는 도보여행을 전면 노출하지 않되, 목적지 케어 상세에 "걷는 시간대 날씨" 정도의 작게 검증 가능한 항목 추가 가능.

### 10. 접근성 민감 사용자

- 상황: 작은 글씨, 낮은 대비, 스크린리더 라벨에 민감함.
- 잘 맞는 점: DOM 기준 주요 버튼에 `aria-label`이 있음. 목적지 검색 입력도 라벨 확인됨.
- 문제/개선: 스크린샷 기준 작은 보조 텍스트와 회색/청색 카드 위 텍스트 대비는 실제 WCAG 수치 검증 전 위험 있음. 시간 선택 UI는 버튼 수가 많아 스크린리더 탐색 부담이 클 수 있음.
- 개선안: 시간 선택은 기본값 요약 + 편집 버튼 구조로 줄이고, 보조 텍스트 대비 수치 검증 필요.

## 공통 강점

- 홈에서 날씨 숫자보다 준비 판단을 앞세우는 방향은 문서 타겟과 맞음.
- 목적지 검색, 선택, 저장, 출발 탭 반영까지 핵심 흐름은 이번 캡처에서 연결됨.
- 알림 권한을 강제하지 않고 `알림 없이 저장`을 제공함.
- MY에서 계정/권한/알림/정책의 진입점이 분리되어 있음.

## 관찰된 문제와 개선 필요

1. 목적지 케어 화면의 시간 선택 UI가 과하게 큼.
   - 근거: `05-destination-care.png`, `06-destination-care-lower.png`
   - 영향: 급한 사용자는 출발 판단보다 설정 조작으로 먼저 인식할 수 있음.
   - 개선: 계산 결과 우선, 시간 수정은 접힌 컨트롤로 이동.

2. 알림 설정 화면의 상태 문구가 많음.
   - 근거: `12-alert-settings.png`
   - 영향: 권한을 켜야 하는지, 예약을 해야 하는지, 확인을 해야 하는지 한 번에 구분하기 어려움.
   - 개선: 권한 미허용 상태에서는 한 단계 CTA만 우선 노출.

3. 저장 완료 피드백이 약함.
   - 근거: `08-saved-without-notification.png`, `09-depart-tab-saved.png`
   - 영향: 사용자는 저장됐는지 출발 탭 이동 전까지 확신하기 어려움.
   - 개선: 저장 직후 완료 배너 또는 목적지 카드 상태 변경 필요.

4. 비 시간의 날짜 맥락이 약함.
   - 근거: `01-first-open.png`, `13-home-after-save.png`
   - 영향: 새벽/밤 시간대는 오늘인지 내일인지 헷갈릴 수 있음.
   - 개선: `오늘`, `내일`, `n시간 후` 중 하나를 함께 표기.

5. 장소 검색 결과의 카테고리 신뢰 위험 있음.
   - 근거: `03-destination-search-results.png`
   - 영향: 검색 결과 자체는 정상이나, 장소 성격과 카테고리 표시가 맞지 않아 보이면 추천 신뢰가 낮아질 수 있음.
   - 개선: 카테고리보다 주소/제공처/거리/행정구역을 우선.

6. 첫 홈에서 수동 위치 변경 affordance가 약함.
   - 근거: `01-first-open.png`
   - 영향: 위치 권한을 꺼둔 사용자는 어디서 위치를 바꾸는지 바로 알기 어려움.
   - 개선: `기본 위치 서울` 영역에 위치 변경 버튼성을 명확히 부여.

## 접근성 위험

- 주요 조작 요소에 접근성 라벨은 있음. 이는 강점임.
- 다만 색 대비, 글자 크기, 스크린리더 실제 읽기 순서, 키보드 탐색은 스크린샷만으로 확정 불가함.
- 시간 선택 UI는 버튼이 많아 보조기기 사용자에게 탐색 비용이 클 가능성 있음.
- 하단 탭이 고정되어 있어 화면 하단 CTA와 겹치는지 여부는 이번 캡처에서는 치명 문제로 보이지 않았으나, 작은 화면/폰트 확대에서는 별도 검증 필요함.

## 증거 한계

- 이번 감사는 web export와 In-app browser 기준임.
- Android/iOS 네이티브 권한 팝업, 실제 푸시 수신, 딥링크, GPS 정확도는 이번 캡처만으로 검증하지 않음.
- 알림 허용 버튼은 실제 권한 전송 행위로 이어질 수 있어 누르지 않음.
- 검색 결과는 현재 API/fixture 응답 기준으로만 판단함.
- 허구 버그는 작성하지 않음. 문제는 화면에서 확인된 UX 위험 또는 개선 필요로만 기록함.

## 우선순위

1. P0: 목적지 케어 화면에서 계산 결과 우선, 시간 선택 접기.
2. P0: 알림 권한 미허용 상태의 다음 행동 단순화.
3. P1: 저장 완료 피드백 추가.
4. P1: 비 시간에 오늘/내일 맥락 추가.
5. P1: 검색 결과 카테고리 신뢰도 개선.
6. P2: 홈 위치 변경 affordance 강화.
