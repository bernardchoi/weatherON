# 소형 사이즈 가독성 검증 (icon-primary)

16px / 24px / 32px 렌더링 결과 (contact_sheet.png 참고)

라이트모드 v2 렌더링 결과는 `contact_sheet_light_v2.png` 참고. `icon-light-v2`는 Navy `#1F4E79`, Accent `#C2410C`, Paper `#F8FBFF` 기준으로 생성했다.

## 결과

- **16px**: 골드 원 주변 3개 광선(rays)이 1px 두께로 거의 사라지며 원과 뭉쳐 보임. 다만 토글 필(크림색) + 구름 + 골드 노브로 구성된 전체 실루엣은 또렷하게 유지되어 "날씨 토글" 형태로 인식 가능. 왕관/촛불 등으로 오인될 위험은 낮음 — 광선 끝이 뾰족하게 돌출되지 않고 노브에 흡수되기 때문.
- **24px**: 광선 3개가 "반짝임/햇살" 형태로 인식 가능. 구름과 토글 노브 간 명도 대비도 안정적.
- **32px 이상**: 광선·구름·토글 디테일 모두 명확.

## 결론

- 16px(파비콘, 앱 리스트 소형 아이콘 등)에서는 광선 디테일에 의존하지 말 것. 핵심 인식 요소는 **토글 필 + 골드 노브 + 구름의 실루엣 조합**이며, 이는 16px에서도 유효함.
- 광선이 사라져도 브랜드 오인(왕관/촛불 등) 위험은 낮음 — 별도 수정 불필요.
- 16px 전용 별도 단순화 버전은 현재 불필요하다고 판단. 단, 추후 실제 OS 파비콘 렌더링(서브픽셀 안티앨리어싱 차이)에서 문제가 발견되면 광선을 제거한 "icon-mono-16-simplified" 버전을 별도 제작 검토.
