# WeatherON · ON Square 캐릭터 아트 바이블 (날씨 크리처 · 대표 + 컴패니언)

> 목적: ON Square 꾸미기(dress-up) 캐릭터를 **고피델리티 일러스트**로 일관되게 양산하기 위한 스타일 락 문서. 일러스트레이터/AI 이미지 생성 어느 쪽이 작화하든 동일한 룩이 나오도록 비율·셰이딩·컬러·표정·슬롯을 고정함.
> 전제: 손코딩 SVG는 작은 UI용 미니멀 글리프 티어로만 유지(2단 피델리티). 히어로·상점·프로필은 이 바이블의 일러스트 티어로 제작.
> 기준 문서: brand/WeatherON_디자인_정체성_가이드.md(서브브랜드 허용) · mockups/WeatherON_S0_mockup.jsx(원소 정의) · design-process/WeatherON_아바타_치비_dressup_시안.html(슬롯 시스템 원형)

## 1. 컨셉 — 친근한 날씨 크리처 (대표 + 컴패니언) (2026-06-21 개정)

추상 "정령"에서 **정붙는 날씨 크리처(동물에 가까운 친근한 형태)**로 전환. 한 마리 한 마리가 구체적 실루엣을 가져 일관성·양산·정붙기·IP화에 유리함(추상 정령은 AI 렌더가 제멋대로 해석하던 문제도 해결). Quiet Horizon 무드는 유지 — 차분·따뜻, 과한 가챠 키치 배제. "조용히 곁을 지키는 날씨 친구" 톤.

**운영 구조 — 대표 1 + 원소 6 컴패니언**:
- **대표 마스코트(Hero)**: 날씨 중립의 브랜드 얼굴. WeatherON/ON Square 자체를 상징(앱 아이콘 짝꿍·스티커·마케팅 얼굴). 한 캐릭터를 제대로 뽑아 강한 IP화 → 2026 "마스코트 주도 브랜드" 트렌드 정렬.
- **원소 6 컴패니언(Solar·Rain·Storm·Wind·Frost·Mist)**: 대표보다 작은 **소형 오브형 크리처 종**. 대표와 골격은 다르지만 얼굴·배 패치·네이비 라인·골드 포인트·셰이딩 문법을 공유한다. 6종끼리는 동일한 컴패니언 골격을 유지하고 원소 컬러 + 모티프 + 오라/표정으로 변주한다.
- **원소 = 일회성 선택이 아니라 수집·체크인 축**: 날씨 체크인으로 컴패니언을 모음(가챠/컬렉션 루프). S0는 대표 마스코트로 즉시 시작하고 6종을 미리보는 구조, S1은 대표를 메인으로 두고 오늘의 동행 컴패니언 XP를 관리하는 구조로 반영 완료.
- **런칭 제작 범위**: ON Square Lite는 대표 마스코트 기본 상태·컴패니언 6종 기본 상태·체크인/XP 반응까지만 필수. 의상·펫·이펙트·상점용 변형은 출시 후 v1.1 제작 범위로 분리.

핵심 룩 3요소(대표·컴패니언 전부 공통, 절대 불변):
- **네이비 아웃라인** `#15294D` — 가챠 특유의 굵은 검정 외곽선 대신 네이비. 절제 유지.
- **가슴 골드 젬 + 미니 광선** — 앱 아이콘/워드마크 ON 스위치 모티프 계승(공유 글리프).
- **골드 림라이트** — 머리/어깨 상단 가장자리에 얇은 골드 빛(Quiet Horizon "첫 빛" 차용).

> 변경 메모(2026-06-22): 1.8등신은 대표 마스코트 전용이다. 컴패니언은 대표 키의 70~75%인 단일 오브형 골격을 사용한다. 8슬롯의 이름과 레이어 순서는 공유하되 대표/컴패니언별 앵커 좌표 템플릿을 분리한다.

## 2. 프로포션 — 대표/컴패니언 이중 골격 (2026-06-22 개정)

### 2.1 대표 마스코트

**1.8등신 치비(확정).** 큰 둥근 머리 + 분리된 작은 몸통 + 스터비 팔다리. 머리 비중이 큼(머리:전신 ≈ 1:1.8).

> **귀여움 다이얼 결정값**: "B 균형의 얼굴(중간 눈·하이라이트 2점·적당 볼터치) + C의 등신비(1.8)". 실루엣은 귀엽게(머리 큼), 눈·볼터치는 절제 유지 → 시장 귀여움과 브랜드 절제의 절충. 근거 시안: design-process/WeatherON_아바타_다이얼_B등신C.html

| 항목 | 비율(머리=1H 기준) | 메모 |
|---|---|---|
| 전체 키 | **1.8H** | 머리 1 + 몸통·다리 0.8 (머리 비중 큼) |
| 머리 폭 | 1.0H | 둥근 사각(연속 곡률), 광대 부드럽게 |
| 눈 크기 | 머리폭의 **0.26** | B 균형값 — 중간~큰 둥근 눈(절제값 0.22에서 상향). 좌우 눈 간격 = 눈 1개 폭 |
| 몸통 | 0.55H 폭 / 0.5H 높이 | 작은 빈백 실루엣 |
| 팔/다리 | 짧고 둥근 캡슐 | 손가락·발가락 디테일 최소 |

캐릭터는 정면 기준이되 약간의 3/4 틀기 허용(생동감). 정수리·턱·어깨가 부드러운 곡선으로 연결.

> 운용 메모: 머리 비중이 커 작은 UI(순위·칩)에선 몸·아이템 노출 면적이 줄어듦. 해당 영역은 미니멀 글리프 티어(10장)로 대체하므로 1.8 유지에 문제 없음.

### 2.2 원소 컴패니언 6종

**소형 단일 오브형(확정).** 머리와 몸통이 분리되지 않은 둥근 콩/알 형태이며, 대표와 나란히 배치할 때 전체 키는 대표의 **70~75%(기준 72%)**다. 6종 모두 동일한 외곽 실루엣·눈·배 패치·팔다리 위치를 유지한다.

| 항목 | 컴패니언 기준 | 메모 |
|---|---|---|
| 전체 키 | 대표의 **0.72배** | 오라/파티클 제외 본체 기준 |
| 본체 비율 | 폭:높이 ≈ **0.82:1** | 둥근 세로 오브형, 머리/목 분할 없음 |
| 얼굴 영역 | 본체 상단 42% | 눈 중심선은 전체 높이의 약 36% |
| 눈 크기 | 본체 폭의 **0.14** | 대표보다 단순하고 작은 UI에서도 판독 가능 |
| 배 패치 | 본체 폭의 **0.48~0.52** | 크림색 공통, 원소색 본체와 명확히 분리 |
| 팔/다리 | 짧은 물방울 캡슐 | 손가락·발가락 없음, 팔끝 골드 팁 유지 |

**위계 규칙:** 대표는 머리 노브·가슴 젬·분리형 몸통으로 “안내자”처럼 보이고, 컴패니언은 작은 오브형과 풍부한 원소 오라로 “곁을 따라다니는 수집 생명체”처럼 보여야 한다. 대표를 단순 축소해 컴패니언으로 만들거나, 컴패니언을 1.8등신으로 늘리지 않는다.

**공유 패밀리 DNA(골격보다 우선):** 네이비 아웃라인 `#15294D` · 크림 얼굴/배 패치 · 둥근 네이비 눈과 캐치라이트 1점 · 잔잔한 미소 · 짧은 팔다리 · 팔끝 골드 팁 · 동일한 소프트 셀셰이딩과 상단 림라이트.

## 3. 페이스 시스템

"차분하지만 친근(calm but friendly)". 큰 둥근 눈 + **캐치라이트 1점이 기본값(확정)**, 표정은 과장 금지.

- **눈(확정)**: 둥근 형태(머리폭 대비 0.26), 짙은 네이비 동공 + **화이트 캐치라이트 1점(안광 절제, 매트)** → "살짝 맹" 결. 하이라이트 2점·별 글린트는 평상시 금지, **Living Mascot "맑음/기쁨" 기분·레어 드랍 등 강조 모먼트에만** 허용.
- **입(확정)**: **입꼬리 올린 잔잔한 미소가 기본**(입꼬리 내림 금지). 폭은 양 눈 안쪽 사이.
- **볼터치**: 대표(캐논)는 **옅게**만. 원소 컴패니언은 원소 light 톤으로 옅게. 과한 발그레는 지양.
- **표정 변형 세트(기본 6종)**: 기본(은은한 미소) · 기쁨(입 벌린 웃음) · 윙크 · 놀람 · 나른(반쯤 감김) · 시무룩. 두 용도로 쓰임 — ① 시스템 자동(Living Mascot 기분, 7장) ② 사용자 커스터마이즈(FACE 슬롯, 6.1). 눈썹·눈·입 3파츠를 분리 제작해 조합 확장 가능.

원소별 성격 톤(표정·자세에 반영): Solar 밝음, Rain 차분·말간, Storm 장난기, Wind 활발, Frost 살짝 도도·시원, Mist 졸린 듯 나른.

## 5. 캐릭터 디자인 — 대표 + 원소 6 컴패니언

전 캐릭터 공통(네이비 아웃라인·가슴 골드 젬·골드 림라이트·차분 친근 페이스·동물적 친근 요소) 위에 정체성 부여.

### 5.0 대표 마스코트 (Hero) — 확정 캐논 (2026-06-21 LOCK)

**확정 캐논 레퍼런스(LOCK): `brand/ON Square 마스코트.png`** — 사용자 최종 선정 원본. **앱 프로덕션 누끼 에셋: `brand/ON Square 대표마스코트_기본_투명-v1.png`**. 얼굴(입꼬리 올린 미소·살짝 맹)·머리 골드 노브·몸통 디테일을 유지하며, 앱·목업에서는 배경이 합쳐진 원본 대신 투명 PNG를 사용한다. 핸드오프 문서: brand/WeatherON_ONSquare_대표마스코트_캐논.html

**제작 기준 시트(2026-06-22 확정)**:
- `brand/ON Square 대표마스코트_턴어라운드·표정_기준시트-v2.png` — 상단 정면·3/4·측면·후면, 하단 기본·기쁨·걱정·놀람·졸림·결의. 걱정 상태도 입꼬리는 아래로 내리지 않음.
- `brand/ON Square 대표마스코트_날씨반응·모션_기준시트-v2.png` — 상단 맑음·흐림·비·폭염·한파/눈·폭풍·밤, 하단 idle·체크인 점프·박수·보상 발견/윙크.
- 기준 시트는 캐논을 대체하지 않음. 단일 캐논의 회전·표정·날씨 이펙트·키포즈를 구현하기 위한 파생 제작 규격임.

확정 스펙(이 외 변경 금지):
- **실루엣**: 라운드 블롭(푸딩형). **고양이 귀·수염 없음**(순수 블롭).
- **2톤 바디**: 위 = 크림/오프화이트 둥근 돔 머리, 아래 = 진한 네이비 하체 + 가운데 둥근 크림 배 패치.
- **골드 포인트(글리프 캐릭터화)**: 머리 위 **골드 "태양노브" 한 점** + 배 패치 중앙 **골드 물방울 젬** + 작은 팔끝 골드 팁. ON 스위치 모티프 계승.
- **얼굴(resting)**: 큰 둥근 네이비 눈 + **캐치라이트 1점(안광 절제, 매트)** → "살짝 맹 · 은은한 차분함". **입은 입꼬리 올린 잔잔한 미소**(절대 입꼬리 내리지 않음). 볼터치는 이 모델 수준의 옅은 정도로만(과하지 않게).
- **아웃라인/셰이딩**: 네이비 `#15294D` 아웃라인 + 소프트 셀셰이딩 + 머리 골드 림라이트.

### 5.1 원소 6 컴패니언 — 소형 오브형 패밀리 (방향 확정 2026-06-22)

**대표 캐논과 디자인 DNA를 공유하되 골격은 분리한다.** 컴패니언 6종끼리는 2.2절의 소형 단일 오브형 골격을 공통 캐논으로 사용한다. 공통 고정: 네이비 아웃라인 + 크림 얼굴/배 패치 + 짧은 팔다리 + 팔끝 골드 팁 + 캐치라이트 1점 + 입꼬리 올린 미소.

- **실루엣**: 대표의 머리·몸 분리를 복제하지 않고, 머리와 몸이 이어진 **둥근 세로 오브형**을 유지한다.
- **원소 표현은 4축이 아니라 "오라/파티클 + 표정·포즈 성격" 축으로 강화**(텍스처·소품 배제). 단순 색+아이콘 1차원 표현 금지 — 각 원소를 **주변 이펙트 + 개성 있는 표정/포즈**로 살림.

| 원소 | 하체·젬 컬러 | 머리 모티프 | 오라/파티클 | 표정·포즈 |
|---|---|---|---|---|
| **Solar** | Gold `#F0A020` | 태양 광선 | 황금 글로우 + 반짝이 입자 | 활짝·활기찬 |
| **Rain** | Sky `#4A8FD4` | 물방울 | 작은 비구름 + 떨어지는 빗방울 | 차분·말간 |

## 6. 드레스업 슬롯 시스템

아이템은 정의된 슬롯에만 부착 → 수백 개 아이템도 비율이 어긋나지 않음. 슬롯 이름과 레이어 순서는 대표·컴패니언이 공유하지만, 골격이 다르므로 **앵커 좌표 템플릿은 `HERO`와 `COMPANION_ORB` 2종으로 분리**한다. 아이템 원화도 필요한 경우 두 체형용 변형을 함께 제작한다. 레이어 순서(뒤→앞): 배경 → 오라 → 캐릭터 베이스 → 의상 → 목 → 손소품 → 얼굴 → 머리.

| # | 슬롯 | 용도 |
|---|---|---|
| 1 | HEAD_TOP | 모자·왕관·날씨캡(원소 모티프 위/대체) |
| 2 | FACE | 안경·표정 교체·마스크 |
| 3 | NECK | 머플러·목걸이 |
| 4 | BODY | 의상·코스튬 |
| 5 | HAND_L | 든 소품(좌) |
| 6 | HAND_R | 든 소품(우)·우산 |
| 7 | AURA / FX | 오라·파티클 이펙트(레어 기상 드랍 등) |
| 8 | SCENE / BG | 배경·프레임 |

### 6.0 HERO 앵커 좌표 v1

캐릭터 바운딩박스 좌상단을 `(0,0)`, 우하단을 `(100,100)`으로 본 정규화 좌표. 실제 리깅 시 기준 시트 정면을 1차 템플릿으로 사용.

| 슬롯 | X | Y | 기준 |
|---|---:|---:|---|
| HEAD_TOP | 50 | 3 | 골드 태양노브 중심 |
| FACE | 50 | 31 | 두 눈 사이 중심 |
| NECK | 50 | 54 | 크림 머리와 네이비 몸 경계 |
| BODY | 50 | 72 | 배 패치 중심 |
| HAND_L | 22 | 69 | 화면 기준 왼팔 골드 팁 |
| HAND_R | 78 | 69 | 화면 기준 오른팔 골드 팁 |
| AURA / FX | 50 | 48 | 전신 외곽 기준 중앙 |
| SCENE / BG | 50 | 50 | 캐릭터와 독립된 전체 캔버스 중앙 |

좌우 반전 애니메이션에서는 HAND_L/HAND_R의 화면 기준 좌표를 스왑함. 노브·얼굴·배 패치·팔끝 골드 팁은 본체 고정 파츠이므로 의상/이펙트가 가리지 않게 함.

아이템 제작 규칙: 락된 캐릭터 비율·관절 위치에 맞춰 작화, 팔레트는 자유(단 채도/명도는 캐릭터와 충돌하지 않게), 아웃라인 두께·셀셰이딩 방식은 베이스와 동일하게 유지.

### 6.1 표정·감정 커스터마이즈 (FACE 슬롯 확장)

2026 꾸미기 트렌드의 핵심 축은 "감정 표현 커스터마이즈"임. FACE 슬롯을 액세서리(안경·마스크) 전용에서 **표정 파츠 교체**까지 확장함.

- **3파츠 분리**: 눈썹 / 눈 / 입을 각각 교체 가능한 파츠로 제작 → 조합으로 표정 다양화.
- **표정 프리셋 상품화**: 기본 6종(3장) 외에 시즌·이벤트 표정을 상점 아이템으로 판매(예: 하트눈, 별눈, 졸린눈).
- **시스템 표정과 충돌 방지**: 사용자가 고정 표정을 선택하면 Living Mascot 자동 기분(7장)은 "강조 오버레이"(볼터치·이펙트)만 적용하고 표정 자체는 사용자 선택을 우선.
- **제작 규칙**: 눈 파츠는 눈 형태 그리드(눈폭=머리폭 **0.26**, 간격=눈 1개 폭, 하이라이트 2점)에 정렬. 귀여움 다이얼 확정값(2장: 1.8등신 + B 균형 얼굴) 기준.

## 7. Living Mascot — 모션 · 반응 · 날씨 연동 기분

2026 마스코트 트렌드의 승부처는 "정적 캐릭터 → 살아 반응하는 캐릭터(Living Brand)"임. 정령은 그려진 그림이 아니라 **오늘 날씨에 반응하는 친구**여야 함 — 이게 날씨앱만이 가질 수 있는 차별점이며 ON Square의 핵심 후크.

### 7.1 모션 레이어 (전 정령 공통)
- **Base Idle — 상시 루프**: 3.2초 주기 호흡(전체 Y ±1.5%, 몸통 Scale Y 0.99↔1.01) + 4~7초 불규칙 눈 깜빡임 + 머리 노브 좌우 2°. 정지 상태에서도 "살아있음"만 느껴지는 강도로 제한.
- **Weather State — 지속 상태**: `clear / cloud / rain / heat / cold / storm / night` 중 하나. 표정·포즈 프리셋과 AURA/FX를 교체하며 Base Idle 위에 합성.
- **Event One-shot — 1회 재생**: `checkIn / missionComplete / rareReward` 이벤트에 각각 체크인 점프·미션 달성 반응·윙크/파티클을 재생한 뒤 현재 Weather State로 복귀.
- **Mission Complete 반응 세트**: 미션 달성은 단일 모션으로 고정하지 않고 `jump / clap / thumbsUp` 3종을 사용한다. 가벼운 일일 미션은 작은 점프, 누적·연속 달성은 박수, 난이도 높은 미션이나 사용자가 직접 완료한 성취는 엄지손가락을 내미는 동작을 우선 적용한다. 반복 피로도를 줄이기 위해 동일 미션 타입이 연속되면 3종을 순환.
- **날씨 소품 유지**: 이벤트 반응 중에도 현재 Weather State의 핵심 소품과 무드는 유지한다. 비 상태의 우산·빗방울, 한파의 머플러·눈송이, 밤의 달/별 아우라, 폭풍의 구름·번개가 이벤트 PNG로 전환되며 사라지면 안 된다. 조합 에셋이 없는 경우에만 기본 Event PNG를 임시 fallback으로 사용.
- **등장/선택**: 스프링 오버슈트(정체성 가이드의 `cubic-bezier(0.34,1.56,0.64,1)`)로 톡 튀어 등장.
- **우선순위**: `rareReward > missionComplete > checkIn > weather > idle`. 이벤트 중 새 이벤트는 큐에 1개만 보관하고 중복 탭은 무시.
- **과한 액션 금지**: 상시 루프 이동폭은 캐릭터 키의 2% 이하, 날씨 흔들림은 3° 이하, 이벤트 점프는 키의 7% 이하. 박수는 2회 이하, 엄지 동작은 1회 내밀기 + 짧은 고개 끄덕임까지만 허용. 화면 전체 이동·연속 회전·무한 점프 금지.

### 7.2 날씨 데이터 연동 기분 (자동 표정/포즈)
사용자의 실제 오늘 날씨를 정령 기분으로 매핑(앱의 H1 날씨 상태 토큰과 1:1). 사용자가 FACE 슬롯에서 고정 표정을 고르면 표정은 유지하고 볼터치·이펙트만 덧입힘(6.1 규칙).

| 오늘 날씨 | 기분/표정 | 포즈·이펙트 |
|---|---|---|
| 맑음 | 밝은 미소, 눈 반짝 | 가벼운 점프, 골드 글로우 강화 |
| 흐림 | 차분, 옅은 미소 | 느린 idle, 채도 살짝 down |
| 비 | 말간·차분 | 우산 소품 자동 노출, 빗방울 파티클 |
| 폭염(35°C↑) | 늘어짐, 땀방울 | 부채질 모션, 열기 아지랑이 |
| 한파/눈 | 오들오들, 발그레 | 몸 움츠림, 입김 파티클 |
| 폭풍 | 놀람·긴장 | 바람에 흔들림, 번개 플래시 |
| 밤 | 나른·졸림(반쯤 감긴 눈) | 느린 호흡, 별 파티클 |

> 구현 노트: 제작 포맷은 **Rive 우선**. `Base / Weather / Event` 3개 레이어 State Machine으로 구성하고 앱 입력값 `weather`, `event`, `missionAction`, `reducedMotion`, `isVisible`을 연결한다. 캐릭터 본체는 파츠 분리 원본 1벌만 작화하고 표정 파츠(6.1) 스왑 + 이펙트 레이어 조합으로 양산한다. Rive 적용 전 검증은 CSS transform 프로토타입으로 진행한다.
> 시각 기준: `brand/ON Square 대표마스코트_날씨반응·모션_기준시트-v2.png`. 이펙트는 AURA/FX 분리 레이어로 제작하고 본체에 합치지 않음.
> 앱 상태 에셋: `brand/ON Square 대표마스코트_날씨-{clear|cloud|rain|heat|cold|storm|night}-v1.png`. 모두 투명 배경 PNG이며 Weather State 입력에 따라 본체 이미지를 교체한다.
> 앱 이벤트 에셋: `brand/ON Square 대표마스코트_이벤트-{idle|mission-jump|mission-clap|mission-thumbsUp}-v1.png`. 기준시트 두번째 줄에서 분리한 투명 배경 PNG이며 Event 입력 시 잠깐 표시한 뒤 현재 Weather State 이미지로 복귀한다.
> 앱 날씨+이벤트 조합 에셋: `brand/ON Square 대표마스코트_날씨이벤트-{rain-mission-jump|rain-mission-clap|rain-mission-thumbsUp|cold-mission-clap|night-mission-clap|storm-mission-thumbsUp}-v1.png`. 우산/머플러/밤 아우라/폭풍 이펙트처럼 상태 연속성이 깨지기 쉬운 조합부터 우선 제작한다.
> 성능·접근성: 화면 밖·백그라운드에서는 루프 정지. `prefers-reduced-motion` 또는 `reducedMotion=true`에서는 호흡·날씨 이동을 정지하고 이벤트를 150ms 젬 발광/표정 교체로 축약. 30fps 기준, 동시 파티클 24개 이하.
> 개발 핸드오프: `brand/WeatherON_ONSquare_대표마스코트_Rive_모션_핸드오프.md` · 동작 검증: `brand/WeatherON_ONSquare_대표마스코트_2D모션_프로토타입.html`.
