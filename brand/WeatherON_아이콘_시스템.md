# WeatherON 아이콘 시스템

> 이 문서는 두 가지를 다룬다: **(A) 인앱 UI 아이콘**(탭바·기능·시스템 아이콘) — 지금까지 목업마다 산발적으로 써온 패턴을 공식 규칙으로 고정한다. **(B) 런처 앱 아이콘 운용** — 비주얼 자체는 `brand/weatheron_ci_bi.html`에 이미 확정돼 있으므로, 여기서는 "어떤 변형을 언제 쓰는지" 매핑과 빠진 부분만 보강한다.

---

## A. 인앱 UI 아이콘 시스템 (신규 정립)

### 1. 그리드 & 드로잉 규칙

- **뷰박스**: 24×24 기준선에서 그린다 (Feather/Lucide 계열 라인 아이콘 컨벤션, MIT 라이선스 — 새 아이콘을 추가할 때도 이 컨벤션을 따르거나 같은 스타일로 직접 그린다).
- **선**: `strokeWidth 1.8`, `strokeLinecap round`, `strokeLinejoin round`, 기본 `fill none`. 면 채우기는 날씨 상태 아이콘(해의 원반 등) 등 예외적인 경우에만 허용.
- **코너 처리**: 별도 코너 라운딩 파라미터 없음 — 24px 그리드 안에서 path 자체가 라운드 캡으로 둥글게 마감된다.

### 2. 사이즈 스케일

| 용도 | 크기 |
|---|---|
| 탭바 아이콘 | 21px |
| 리스트 행 보조 아이콘 (날씨 상세 메트릭, 알림 카드 등) | 18px |
| 헤더/원형 버튼 내부 아이콘 (뒤로가기·공유·검색) | 16px |
| 쉐브런·보조 인디케이터 | 14px |

이 4단 스케일 밖의 임의 크기는 쓰지 않는다 — 화면마다 아이콘 비례가 미세하게 달라 보이는 것을 방지.

### 3. 컬러 적용 규칙

- **기본(중성) 아이콘** — 네비게이션, 액션 버튼, 보조 표시: `currentColor` 또는 `INK(a)` (`rgba(232,237,246,a)`) / `MIST` (`#6B80A0`).
- **날씨 상태를 표현하는 아이콘**(해·구름·비·달 등)에는 항상 기능색을 입힌다: 다크모드는 해=`GOLD`(#F4B63F), 비/흐림=`SKY`(#4AA3DF) 또는 중성 Mist, 청량/신선=`CLEAR`(#2FC6A3)을 쓴다. 라이트모드는 흰 카드 위에서 흐려 보이지 않도록 `GOLD`(#C2410C), `SKY`(#237BBD), `CLEAR`(#007F73), `WARM`(#C84A2F)로 리틴트한다.
- **탭바**: 활성 탭 아이콘은 `GOLD`, 비활성은 `rgba(168,196,224,0.70)` — AA 대비 기준 통과를 위해 0.55에서 상향(2026-06-20, `WeatherON_design_system.jsx` 동기화).
- **골드는 "ON/활성" 의미 전용** — CI/BI 가이드의 "골드는 CTA 전용, 배지·아이콘에 분산 사용 금지" 원칙을 UI 아이콘에도 동일하게 적용한다. 임의 장식 아이콘에 골드를 쓰지 않는다.

### 4. 인터랙션

- 아이콘이 단독으로 탭 가능한 영역(아이콘 버튼, 탭바 항목)은 항상 상태 레이어 프레스(`PressTintOverlay`, opacity 0→0.12, 120ms)를 적용한다.
- Material 리플의 원형 확산 애니메이션이나 별도 스케일/바운스 애니메이션은 쓰지 않는다 — `brand/WeatherON_디자인_정체성_가이드.md` 4-1장과 동일 원칙.

### 5. 브랜드 글리프와의 경계

- 위치 핀, 화살표, 검색 등 범용 UI 아이콘은 토글+태양노브 모티프(공유 글리프)를 변형해서 만들지 않는다. 그 모티프는 앱 아이콘·워드마크 전용으로 고정한다 (디자인 정체성 가이드 2장 "변하지 않는 축").

### 6. 탭바 아이콘 확정 세트 (5개)

홈 / 코디 / 출발 / MY / 소셜 — 5개 아이콘의 line-art는 이미 모든 목업(H1~H5, C1, G1, M1, 날씨상세)에 동일한 path로 고정되어 있다. 새 화면을 만들 때도 이 path를 그대로 재사용하고, 임의로 다른 아이콘 세트에서 복사해오지 않는다.

### 7. 목적지·장소 카테고리 아이콘

O6/P1/P2/P3의 목적지 라벨은 운영체제별로 형태가 달라지는 이모지 대신 24×24 라인 SVG 아이콘을 사용한다. 회사, 학교, 공항, 숙소, 야구장, 등산, 해변, 계절, 문화 아이콘은 `mockups/WeatherON_destination_icons.jsx`에서 관리하고, 위 그리드·선 굵기·컬러 규칙을 따른다.

목적지 아이콘은 의미 전달용 보조 아이콘이므로 CTA 골드 남용을 피한다. 활성/선택 상태에서는 부모 칩의 텍스트 컬러를 상속하고, 장소 카드에서는 카테고리 상태색(CLEAR/WARM/SKY 등)을 제한적으로 사용한다.

## C. Outfit Spec 기법 (코디 추천 시각화, 2026-06-20)

코디 추천(H1 홈 카드, C1 코디 메인)을 캐릭터/아바타 없이 보여주는 전용 기법. 패션 테크팩의 **플랫 스케치**(입지 않은 평면 도식)와 엔지니어링 **익스플로디드 뷰**(부품별 리더라인 콜아웃)를 결합 — 날씨·코디 앱 카테고리에서 선례가 없는 독자적 패턴이므로, 확장할 때 아래 규칙을 지켜 일관성을 유지한다.

**사이즈별 표현 강도 — 풀사이즈와 압축판은 디테일을 다르게 줄인다.**

| | 풀사이즈 (C1 메인) | 압축판 (H1 홈 카드 등) |
|---|---|---|
| 블루프린트 그리드 | 사용 (`rgba(232,237,246,0.05)`, 20px 간격) | 사용 안 함 — 작은 사이즈에선 노이즈가 됨 |
| 선 굵기 | 1.6px | 2.2px (작을수록 두껍게 — 작은 사이즈에서 얇은 선은 흐려 보임) |
| 콜아웃 | 카테고리 3행 전부 (번호+리더라인+한글 라벨) | 리더 마커 1개만 (라벨 텍스트 생략) |
| 옷 색 채움 | 실제 색상 100% (베이지·슬레이트 등) | 카드 accent 1색만 (예: CLEAR 틴트) |

**리더 마커는 채워진 원이 아니라 십자 틱(crosshair tick)을 쓴다.** 채워진 원(`<circle fill={GOLD}/>`)은 브랜드의 CTA·활성 탭과 형태가 같아서 "탭 가능한 버튼"처럼 보이는데, 이 마커는 실제로는 탭이 안 되는 순수 라벨이다 — 형태 자체로 "측정점 표기"임을 구분하기 위해 작은 `+` 모양 틱(`LeaderMark`/동일 패턴)을 표준으로 쓴다.

**압축판의 보조 콜아웃은 카드 본문 텍스트와 중복되지 않는 정보를 담는다.** 예: 카드에 이미 "일교차 8°C"가 있으면 글리프 쪽 콜아웃은 같은 숫자를 반복하지 않고 "추천 확정" 체크 같은 별도 신호를 준다.

**옷 색상은 브랜드/기능색과 분리된 독립 팔레트를 쓴다.** 베이지·슬레이트·크림 등 실제 패션 색상만 사용하고, Gold/Sky/Clear 같은 브랜드·기능색 토큰은 리더라인·마커·헤더 readout 같은 "주석" 요소에만 쓴다 — 옷(콘텐츠)과 주석(메타데이터)을 색으로 구분.

## B. 런처 앱 아이콘 운용 (보강)

비주얼 정의(G-1 구성: 단색 NAVY 배경, 크림 토글 트랙, 골드 노브+광선, 페이퍼 구름)와 Liquid Glass 3-레이어(bg/mid/fg) Icon Composer 세팅은 `brand/weatheron_ci_bi.html`에 이미 정의되어 있다. 아래는 4개 변형을 "언제 쓰는지"에 대한 요약 매핑이다.

| 변형 | 구성 | 쓰는 곳 |
|---|---|---|
| `icon-primary` | NAVY bg · 크림 트랙 · 페이퍼 구름 | 기본 앱 아이콘, 스플래시, 라이트 모드 홈 화면, 소셜 프로필 |
| `icon-light-v2` | 밝은 배경용 Clear Weather UI 톤 · Navy `#1F4E79` · Dawn Orange `#C2410C` | 라이트 배경 위 단독 배치(문서·웹 파비콘 등) |
| `icon-light` | 기존 밝은 배경용 톤 | 구버전 호환용. 신규 라이트 UI에는 `icon-light-v2` 우선 |
| `icon-dark` | 딥네이비 bg · 톤다운 트랙/구름 | 다크 모드 홈 화면 (눈부심 감소) |
| `icon-mono` | 단색 한 가지 톤 | 단색 컨텍스트 — 워치 컴플리케이션, 모노크롬 배지 등 |

에셋은 `assets/icon/`에 16~1024px로 이미 export되어 있다 (`icon-{primary,light,dark,mono}-{16…1024}.png` + 각 SVG 원본). 라이트 v2는 `icon-light-v2-{16…1024}.png`와 `icon-light-v2.svg`를 사용한다.

### Android Adaptive Icon (제작 완료, 2026-06-20)

`design-process/scripts/gen_icon_v18_android.py`로 G-1 지오메트리(`gen_icon_v16.render_icon`)를 그대로 재사용해 생성. 에셋: `assets/icon/android/`.

| 레이어 | 파일 | 구성 |
|---|---|---|
| Foreground | `icon-adaptive-foreground-1024.png` | 토글 트랙(크림)+바(NAVY 펀치)+골드 노브·광선+페이퍼 구름, 투명배경 |
| Background | `icon-adaptive-background-1024.png` | NAVY 단색, full bleed, 투명도 없음 |

- 캔버스는 **108dp**를 기준으로 한 1024px 마스터. 콘텐츠는 **세이프존(중앙 66dp 원)** 비율(66/108)에 맞춰 자동 크롭 후 스케일·중앙 정렬 — 어떤 런처 마스크(원형/둥근사각형/티어드롭 등)로 잘려도 토글·구름 실루엣이 클리핑되지 않는다.
- 원형·둥근정사각 마스크 합성 QA는 `WeatherON_adaptive_icon_preview.png`로 확인 — 두 마스크 모두에서 클리핑 없이 정상 노출됨.
- 실제 Android 빌드 시: `mipmap-anydpi-v26/ic_launcher.xml`에서 두 PNG를 각각 `<foreground>`/`<background>`로 지정하고, 필요한 밀도(mdpi~xxxhdpi)별로 다운스케일 export.

### Android 알림 아이콘(notification small icon) (제작 완료, 2026-06-20)

같은 스크립트에서 생성. 에셋: `assets/icon/android/icon-notification-{density}-{px}.png` (mdpi 24 / hdpi 36 / xhdpi 48 / xxhdpi 72 / xxxhdpi 96) + 마스터 `icon-notification-master-432.png`.

- **흰색 단색 실루엣 + 투명배경** — Android 시스템 규격(다른 색상 사용 금지, 시스템이 알림 컨텍스트에 맞게 자체 틴트)을 그대로 따른다.
- "바(off 인디케이터)"는 페인트가 아니라 **진짜 투명 펀치**로 처리 — 흰 트랙 실루엣에 알파 0인 구멍을 뚫어, 단색 제약 안에서도 토글의 "홈" 디테일이 유지된다.
- 16px 가독성 검증(`assets/icon/legibility/README.md`) 결론과 동일하게, mdpi(24px) 기준에서도 광선은 거의 사라지지만 토글 필+노브+구름 실루엣 조합은 또렷이 유지됨을 확인.
